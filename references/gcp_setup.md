# Google Cloud セットアップ手順（ユーザーガイド用）

ユーザーのブラウザ操作をガイドするための正本。1ステップずつ進め、完了報告を待ってから次へ。
Google Cloud Console の画面文言はアップデートで多少変わることがある。ユーザーの報告と
食い違ったら、目的（何を作ろうとしているか）を伝えて画面上の近い選択肢を探してもらう。

## 前提の確認（最初に聞く）

- YouTubeチャンネルを管理しているGoogleアカウントはどれか
  （チャンネルが「ブランドアカウント」の場合も、管理者の個人Googleアカウントで作業する）
- そのアカウントでYouTubeの電話番号確認が済んでいるか
  （カスタムサムネイル設定に必須。未確認なら https://www.youtube.com/verify で先に済ませる）

## STEP 1: GCPプロジェクト作成

1. https://console.cloud.google.com/ にチャンネル管理アカウントでログイン
2. 初回は利用規約に同意（無料。このスキルの用途では課金は発生しない）
3. 「$300分の無料クレジット」「無料トライアル」の案内が出たら**スキップ（閉じる）**。
   トライアルに進むとクレジットカード入力を求められるが、このスキルの用途では
   トライアルもカード登録も一切不要。閉じてそのまま作業できる
4. 画面上部のプロジェクト選択 →「新しいプロジェクト」→ 名前は `youtube-connect` など → 作成

## STEP 2: APIを2つ有効化

「APIとサービス」→「ライブラリ」で以下を検索し、それぞれ「有効にする」:

1. **YouTube Data API v3** … アップロード・概要欄・サムネ用
2. **YouTube Analytics API** … 再生数・視聴維持率などの分析用

片方だけだと後でエラー（accessNotConfigured）になるので、必ず両方。

## STEP 3: OAuth同意画面の設定

「APIとサービス」→「OAuth同意画面」:

1. User Type は「**外部（External）**」を選択（組織のGoogle Workspaceでない限りこれしか選べない）
2. アプリ名（例: youtube-connect）、ユーザーサポートメール、デベロッパー連絡先に自分のメールを入力
3. スコープの追加画面は**スキップしてよい**（認証スクリプト側で要求するため）
4. テストユーザーに自分のGmailを追加
5. 作成完了後、「公開ステータス」を確認し、**「アプリを公開」を押して本番環境に移行する**

### なぜ本番環境に移行するのか（重要）

テスト中（Testing）のままだと、発行したトークンが**7日で失効**し、毎週ブラウザ認証を
やり直すことになる。本番環境（In production）に移行すればトークンは失効しない。

「確認が必要」という案内が出るが、**Google審査は不要**。使うのは自分だけなので、
確認手続きには進まず、そのまま公開状態にしておけばよい。認証時に
「このアプリはGoogleで確認されていません」という警告が出るようになるが、
自分で作ったアプリなので「詳細」→「（安全でないページに）移動」で通過して問題ない。

## STEP 4: OAuthクライアントID作成

「APIとサービス」→「認証情報」→「認証情報を作成」→「OAuthクライアントID」:

1. アプリケーションの種類: **デスクトップアプリ**（ここを間違えると認証が通らない）
2. 名前は任意（例: youtube-connect-desktop）
3. 作成後、**JSONをダウンロード**
4. ダウンロードしたファイルを `~/youtube-connect/client_secret.json` に移動・リネームしてもらう
   （Claudeが `mv` コマンドで代行してよい。Downloadsフォルダのファイル名は
   `client_secret_XXXX.apps.googleusercontent.com.json` のような形式）

## STEP 5 以降

SKILL.md のセットアップモード STEP 2〜4 に戻る（venv作成 → 認証 → 動作確認）。

## APIクォータの説明（ユーザーに伝える）

- YouTube Data API は1日 **10,000ユニット**（プロジェクトごと・毎日リセット）
- 動画アップロード1本 = 約**1,600ユニット**（1日6本程度が上限の目安）
- 概要欄更新 = 50ユニット、情報取得 = 1ユニット、Analytics APIは別枠でほぼ無制限
- 通常運用で困ることはないが、大量一括処理の前には意識する
