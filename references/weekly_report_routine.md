# 週次プロデューサーレポートのルーチン化（自動定期実行）

セットアップ済みのチャンネルに対して、「自チャンネル分析＋競合リサーチ＋次企画の提案」を
定期実行するルーチン（スケジュールタスク）を組む手順。ユーザーが「毎週分析して」
「定期レポートがほしい」「プロデューサーレポートを自動化したい」と言ったらこれを案内する。

## 前提

- clbs-youtube-connect のセットアップが完了していること（`~/youtube-connect/channels.yaml` がある）
- Claude Code のスケジュールタスク（ルーチン）機能を使う。ユーザーが「ルーチン」画面から
  作成する方法と、Claudeがスケジュールタスク作成ツールで代行する方法のどちらでもよい

## 推奨設定

| 項目 | 推奨値 | 理由 |
|---|---|---|
| 頻度 | 週1〜2回（例: 月・木の朝10時） | YouTube Analyticsは確定まで48〜72時間遅れるため、毎日回しても数字が動かない |
| フォルダ | チャンネルの作業フォルダ | レポートの保存先になる |
| 常に許可 | Bash / Read / WebFetch | 分析スクリプト実行と競合リサーチに必要。書き込み系は含めない |

## 指示文テンプレート

以下をユーザーのチャンネルに合わせて穴埋めして使う。`{{ }}` の箇所を必ず置き換えること。

```
あなたはYouTubeチャンネル「{{チャンネル名}}」（{{@ハンドル}}、ジャンル: {{ジャンル・テーマ}}）の
プロデューサー/ディレクターです。本人は制作に専念し、企画・分析はあなたに任されています。
週次プロデューサーレポートを日本語で作成してください。

前提・参照:
- 認証トークン: {{トークンのパス（例: ~/youtube-connect/tokens/main.json）}}
- 分析スクリプト: {{スキルのパス}}/scripts/yt_analytics.py
  （実行は ~/youtube-connect/.venv/bin/python を使う）
- レポート保存先: {{作業フォルダ}}/週次プロデューサーレポート_YYYYMMDD.md

実行内容:
1. 自チャンネル分析: yt_analytics.py で過去7日＋累計の再生数・総再生時間・平均視聴率・
   登録者増減・動画別ランキング・トラフィックソースを取得。前回レポートがあれば読み、
   前週比と変化点を評価。データが未確定（直近2日）の数字は暫定である旨を明記。
2. 競合異常値リサーチ: 同ジャンルの競合チャンネル（{{競合チャンネル3〜5個}}）の直近で
   伸びた動画（そのチャンネルの平均より明らかに再生が多いもの）を WebSearch/WebFetch で調べ、
   フック・タイトル・サムネ・切り口の共通項を抽出。
3. 考察と次企画: 1と2を統合し「次に作るべき動画の企画案3つ（タイトル・フック・
   狙うトラフィック・なぜ今これか）」を提案。自チャンネルで当たった/外した要因の仮説も。
4. レポート構成: ①今週の数値サマリーと判定 ②競合から学んだこと ③次企画3案（推奨1つ明記）
   ④改善提案（あれば）。誇大に断定せず、母数が小さければ限界を明記。

本人が求めているのは「どう考察し→どんな企画にし→どんな結果だったか」が追える透明な
レポートです。プロデューサーとして主体的に判断し、次アクションまで示してください。
```

## セットアップをガイドするときの手順

1. ユーザーに競合チャンネルを3〜5個挙げてもらう（決まっていなければ、チャンネルのジャンルを
   聞いて候補を提案する）
2. テンプレートの `{{ }}` をすべて埋めた指示文を作って見せ、内容を確認してもらう
3. スケジュールタスクとして登録する（頻度・許可ツールは上表のとおり）
4. 初回は「今すぐ実行」で1回走らせ、レポートが正しく生成されるか一緒に確認する

## 運用のコツ（ユーザーに伝える）

- レポートで判明した知見（勝ちパターン・タイトルルール等）は、レポートに書くだけでなく
  メモリやCLAUDE.mdへの追記を提案させると、次回以降の企画精度が上がる
- 登録者数百人未満のうちは週次で数字がほぼ動かないことも多い。その場合は競合リサーチと
  企画提案を主役にし、数字は定点観測として淡々と記録する
- Analytics APIの数字はStudio画面とズレることがある（トラフィックソース別は特に）。
  重要な判断の前にStudioでの確認を促す一文をレポートに入れさせる
