# 週次プロデューサーレポートのルーチン化（自動定期実行）

セットアップ済みのチャンネルに対して、「自チャンネル分析＋競合リサーチ＋次企画の提案」を
定期実行するルーチン（スケジュールタスク）を組む手順。ユーザーが「毎週分析して」
「定期レポートがほしい」「プロデューサーレポートを自動化したい」と言ったらこれを案内する。

## 前提

- clbs-youtube-connect のセットアップが完了していること（`~/youtube-connect/channels.yaml` がある）
- Claude Code のスケジュールタスク（ルーチン）機能を使う。ユーザーが「ルーチン」画面から
  作成する方法と、Claudeがスケジュールタスク作成ツールで代行する方法のどちらでもよい

## 推奨設定

| 項目 | 推奨値 | 理由 |
|---|---|---|
| 頻度 | 週1〜2回（例: 月・木の朝10時） | YouTube Analyticsは確定まで48〜72時間遅れるため、毎日回しても数字が動かない |
| フォルダ | チャンネルの作業フォルダ | レポートの保存先になる |
| 常に許可 | Bash / Read / WebFetch | 分析スクリプト実行と競合リサーチに必要。書き込み系は含めない |

## 指示文テンプレート

以下をユーザーのチャンネルに合わせて穴埋めして使う。`{{ }}` の箇所を必ず置き換えること。

```
あなたはYouTubeチャンネル「{{チャンネル名}}」（{{@ハンドル}}、ジャンル: {{チャンネルのテーマ}}）の
プロデューサー/ディレクター/編集者です。本人は撮影と公開に専念し、企画・分析・改善提案は
あなたに任されています。週次プロデューサーレポートを日本語で作成してください。

前提・参照:
- チャンネル台帳: ~/youtube-connect/channels.yaml（認証トークンのパスはここに書いてある）
- 分析スクリプト: ~/.claude/skills/clbs-youtube-connect/scripts/yt_analytics.py
  （実行は必ず ~/youtube-connect/.venv/bin/python を使う）
- リサーチ資産: {{作業フォルダ}}/research/（clbs-youtube-research の出力置き場）
- 過去レポート: {{作業フォルダ}}/週次プロデューサーレポート_*.md（あれば読み、前週比較に使う）
- レポート保存先: 同じフォルダに 週次プロデューサーレポート_YYYYMMDD.md として保存

実行内容:
1. 自チャンネル分析: yt_analytics.py で過去7日＋累計の再生数・総再生時間・平均視聴率・
   登録者増減・動画別ランキング・トラフィックソースを取得。前回レポートと比較し、
   伸びた動画/落ちた動画と要因仮説を評価。データ未確定（直近2日）なら正直にその旨を書く。
2. 競合異常値リサーチ: clbs-youtube-research スキルの方法論（STEP2「異常値の特定と
   モデリング」）に準拠する。競合チャンネル（{{競合チャンネルを3〜5個}}）の直近で伸びた動画
   ＝そのチャンネルの普段より明らかに再生が多い「異常値」を特定し、フック・タイトル・
   サムネ・切り口をモデリング。research/04_research_summary.yaml があれば読み込み、
   今週の新発見だけを差分追記して更新する（毎回ゼロからやり直さない）。特に重要な1本は
   スキルの _tools/ingest.py で字幕・コメントまで取得し、中身とリテンション設計を解剖してよい。
3. 次企画3案: clbs-youtube-script-pro の企画考案フェーズに準拠する。1と2の材料から、
   常識破壊フレーム＋心理トリガーを核にした精鋭3案（タイトル・サムネ文言・冒頭フック・
   狙うトラフィック・なぜ今これか）を作り、企画審査5項目でセルフ採点して4/5以上の案のみ
   提示。推奨1案を明記。台本までは作らない。レポート末尾に「採用する案が決まったら
   『◯◯案を台本に』と伝えれば clbs-youtube-script-pro が台本化を引き継ぐ」と記す。
4. レポート構成: ①今週の数値サマリーと判定 ②競合から学んだこと（researchの差分）
   ③次企画3案（採点付き・推奨明記）④改善提案（あれば）。誇大に断定せず、
   母数が小さければ限界を明記する。

（clbs-youtube-research / clbs-youtube-script-pro が未導入の環境では、2と3は
 スキルなしの通常リサーチ・通常の企画立案で代替する）

本人が求めているのは「どう考察し→どんな企画にし→どんな結果だったか」が追える透明な
レポートです。プロデューサーとして主体的に判断し、次アクションまで示してください。
```

## 連携スキル（導入済みなら効果が上がる）

- **clbs-youtube-research**: 競合リサーチの方法論と `research/04_research_summary.yaml`
  への蓄積を担う。週次で差分更新すれば競合ナレッジが複利で積み上がる
- **clbs-youtube-script-pro**: 企画の品質ゲート（常識破壊フレーム＋企画審査5項目・
  4/5以上のみ合格）と、採用案の台本化を担う

未導入でもテンプレートはそのまま動く（末尾のフォールバック指示で代替される）。

## セットアップをガイドするときの手順

1. ユーザーに競合チャンネルを3〜5個挙げてもらう（決まっていなければ、チャンネルのジャンルを
   聞いて候補を提案する）
2. テンプレートの `{{ }}` をすべて埋めた指示文を作って見せ、内容を確認してもらう
3. スケジュールタスクとして登録する（頻度・許可ツールは上表のとおり）
4. 初回は「今すぐ実行」で1回走らせ、レポートが正しく生成されるか一緒に確認する

## 運用のコツ（ユーザーに伝える）

- レポートで判明した知見（勝ちパターン・タイトルルール等）は、レポートに書くだけでなく
  メモリやCLAUDE.mdへの追記を提案させると、次回以降の企画精度が上がる
- 登録者数百人未満のうちは週次で数字がほぼ動かないことも多い。その場合は競合リサーチと
  企画提案を主役にし、数字は定点観測として淡々と記録する
- Analytics APIの数字はStudio画面とズレることがある（トラフィックソース別は特に）。
  重要な判断の前にStudioでの確認を促す一文をレポートに入れさせる
