# トラブルシューティング

エラーメッセージ別の対処。スクリプトのエラー出力から該当パターンを探す。

## 認証・トークン系

### `invalid_grant` / `Token has been expired or revoked`
リフレッシュトークンが失効している。原因はほぼ次のどちらか:
1. **OAuth同意画面が「テスト中」のまま** → トークンは7日で失効する仕様。
   GCPコンソールで「アプリを公開」して本番環境に移行してから、`yt_authorize.py` で再認証
2. ユーザーがGoogleアカウント設定でアプリのアクセス権を取り消した → 再認証

### 「このアプリはGoogleで確認されていません」という警告画面
正常。自分で作ったアプリなので「詳細」→「（安全でないページに）移動」で進んでよい。

### `redirect_uri_mismatch` / 認証後にブラウザがエラー
OAuthクライアントの種類が「デスクトップアプリ」以外で作られている。
GCPの「認証情報」でデスクトップアプリとして作り直す。

### `access_denied` / アクセスをブロック
OAuth同意画面がテスト中で、ログインしたアカウントがテストユーザーに入っていない。
テストユーザーに追加するか、本番環境に移行する。

### 認証は通るがチャンネル情報が空（items が無い）
認証時のチャンネル選択画面で、チャンネルではなく個人アカウントを選んでいる。
ブランドアカウントのチャンネルを選び直して再認証。

## API有効化・権限系

### `accessNotConfigured` / `API has not been used in project ...`
そのAPIがGCPプロジェクトで有効化されていない。エラーメッセージ内のURLを開くか、
APIライブラリで「YouTube Data API v3」「YouTube Analytics API」の両方を有効化。
有効化直後は反映まで数分かかることがある。

### `insufficientPermissions` / `Request had insufficient authentication scopes`
トークンのスコープが足りない（古いトークンや別用途のトークンを流用した場合）。
`yt_authorize.py` で4スコープ揃った状態で再発行する。

### サムネイル設定だけ403になる
YouTubeアカウントの**電話番号確認**が未完了。
https://www.youtube.com/verify で確認を済ませてから再実行。

## クォータ・制限系

### `quotaExceeded` / `dailyLimitExceeded`
1日のAPIクォータ（10,000ユニット）を使い切った。太平洋時間0時（日本の午後4〜5時）に
リセットされるので待つ。アップロードは1本1,600ユニット消費する。

### `uploadLimitExceeded`
YouTube側のチャンネルとしてのアップロード上限（API経由は特に新規チャンネルで厳しめ）。
24時間待つ。頻発する場合はチャンネルの高度な機能の有効化状況をStudioで確認。

### アップロードした動画が「ロック済み（非公開）」になる
API経由のアップロードは、OAuthアプリがGoogle未確認の場合にスパム判定されることが稀にある。
Studioから確認を申請できる。頻発する場合はOAuth同意画面のアプリ確認手続きを検討。

## 運用系

### アップロードが途中で止まる/失敗する
`yt_upload.py` はレジューム対応（8MBチャンク）だが、ネットワーク断で例外が出たら
再実行してよい（同じ動画が二重になるので、失敗時はStudioで下書きが残っていないか確認）。

### ABORT: トークンのチャンネルが想定と一致しません
`--expect-handle` の値と、トークンが指すチャンネルが違う。channels.yaml の
handle と token の組み合わせを確認。別チャンネルにアップしようとしている可能性が高いので、
ユーザーに確認してから正しいトークンで再実行する。

### Analytics APIの数字がStudioの画面と合わない
仕様。特にトラフィックソース別は再生急増時に SUBSCRIBER が過剰計上されることがある。
重要な判断はStudio画面を正とし、APIは日次の定点観測用として使う。
また、Analyticsのデータは48〜72時間遅れで確定する（直近2日は暫定値）。
